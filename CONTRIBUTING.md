# Contributing

Thanks for your interest in improving this project.

## Before You Start

- Please check the README first to understand the current node set and repository structure.
- If you want to propose a larger change, opening an issue first is helpful.
- Keep changes focused and easy to review.

## Development Notes

- The custom nodes live in the repository root.
- Data files are stored under `data/`.
- Saved example presets live under `user_presets/`.
- Sample workflows live under `workflows/`.

## Pull Requests

- Use clear commit messages.
- Describe what changed and why.
- If you change behavior, update the README when needed.
- If possible, include a sample workflow, screenshot, or usage example for UI-facing changes.

## Scope

Good contribution targets include:

- bug fixes
- better documentation
- data improvements
- new example workflows
- usability improvements for the ComfyUI nodes

## Be Careful With

- breaking changes to node names or output types
- changes that make existing settings or workflows incompatible
- committing private data, local paths, or personal files
